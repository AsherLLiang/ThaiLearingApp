## ⚠️ 注意事项

### ✅ DO（应该做的）

1. **每次提到项目时引用快照**
   ```
   "基于project-snapshot-V3.md..."
   ```

2. **提供具体要求**
   ```
   不要："创建一个卡片"
   应该："创建CourseCard组件，接收course和progress props，
          使用玻璃态效果，匹配现有设计"
   ```

3. **分步骤开发复杂功能**
   ```
   先：数据结构
   再：Store
   然后：组件
   最后：集成
   ```

4. **重大更改后更新快照**
   ```
   让Claude帮你更新，或手动编辑
   ```

### ❌ DON'T（不应该做的）

1. **不引用快照**
   ```
   ❌ "创建一个用户页面"
   ✅ "基于project-snapshot-V3.md，创建..."
   ```

2. **提示词过于模糊**
   ```
   ❌ "做一个列表"
   ✅ "创建课程列表页面，每个卡片显示标题、等级、进度条"
   ```

3. **完全依赖快照调试复杂Bug**
   ```
   复杂bug需要看真实代码
   快照适合架构级别的问题
   ```

4. **让快照长期过时**
   ```
   每周或重大更改后检查更新
   ```

## 💡 最后的建议

1. **不要害怕尝试**
   - Claude基于快照生成的代码通常很准确
   - 即使有小错误，也易于修复
   - 多尝试不同的提示词

   - 每次重大更改后更新

   - 复杂功能拆分为小任务
   - 每步验证后再进行下一步
   - 降低出错概率

4. **与团队分享**
   - 让所有开发者使用相同的快照
   - 统一代码风格
